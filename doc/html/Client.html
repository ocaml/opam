OPAM client.
val init : Types.repository -> Types.Alias.t -> Types.OCaml_V.t -> int -> unit
Initializes the client a consistent state. init repo alias oversion cores means:
  • repo is the initial repository description,
  • alias the compiler alias
  • oversion is the version of the compiler
  • cores is the number of cores

val list : unit -> unit
Displays all available packages
val info : Types.N.t -> unit
Displays a general summary of a package.
val config : Types.config -> unit
Depending on request, returns options or directories where the package is installed.
val install : Types.N.Set.t -> unit
Installs the given set of packages. Take the global file lock.
val update : unit -> unit
Refresh the available packages. Take the global file lock.
val upgrade : unit -> unit
Finds a consistent state where most of the installed packages are upgraded to their latest version. Take the global file lock.
val upload : Types.upload -> string option -> unit
Upload a package to a remote repository. If repo is None then it will look for the repository associated with the package name. Otherwise, it will look for a repository having the right name. Take the global file lock.
val remove : Types.N.Set.t -> unit
Removes the given set of packages. Take the global file lock.
val remote : Types.remote -> unit
Manage remote repositories. Take the global file lock.
val switch : bool -> Types.Alias.t -> Types.OCaml_V.t -> unit
switch and_clone alias descr switch to an OCaml compiler and clone at the end in case and_clone is true. It creates $opam/$alias if it does not exists by reading the contents of $opam/compilers/$descr.comp. It takes the global file lock.
val compiler_list : unit -> unit
compiler_list list the available compiler descriptions
